# Data Model: Archive Format & Pack Command

**Branch**: `002-archive-format-pack` | **Date**: 2026-03-01

## Entities

### Artifact

Represents a single artifact discovered from a well-known directory. Generated during the pack command's discovery phase and serialized into the manifest's `artifacts` array.

| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| Name | string | Artifact name derived from filename or directory | Kebab-case, 1-64 chars, no consecutive hyphens, cannot start/end with hyphen. Matches `^(?!.*--)[a-z0-9]([a-z0-9-]{0,62}[a-z0-9])?$` |
| Type | ArtifactType (string enum) | One of: `skill`, `prompt`, `command`, `agent`, `agent-instructions`, `mcp-server` | Must be a recognized type |
| Path | string | Relative path from package root to the artifact file or directory | Non-empty. Directories end with `/`. |

**Name derivation rules**:
- Skills: directory name under `skills/` (e.g., `skills/test-writer/` -> `test-writer`)
- File-based types: filename with all extensions stripped (everything from the first `.` onward). `code-review.prompt.md` -> `code-review`, `my-prompt.txt` -> `my-prompt`

### ArtifactType

Enum mapping well-known directories to artifact types.

| Directory | Type value |
|-----------|-----------|
| `skills/` | `skill` |
| `prompts/` | `prompt` |
| `commands/` | `command` |
| `agents/` | `agent` |
| `agent-instructions/` | `agent-instructions` |
| `mcp-servers/` | `mcp-server` |

### PackageManifest (extended)

The existing `PackageManifest` struct from 001, extended with the `Artifacts` field.

| Field | Type | JSON key | Required | Notes |
|-------|------|----------|----------|-------|
| SpecVersion | int | `specVersion` | Yes | Always `1` |
| Name | string | `name` | Yes | Scoped name: `@scope/package-name` |
| Version | string | `version` | Yes | Semver: `MAJOR.MINOR.PATCH` |
| Description | string | `description` | No | Max 255 chars |
| License | string | `license` | No | SPDX identifier |
| Artifacts | []Artifact | `artifacts` | No | Generated by pack. `omitempty` so absent when not set. |

### SkillFrontmatter

Parsed from the YAML frontmatter block in `SKILL.md` files.

| Field | Type | YAML key | Required | Validation |
|-------|------|----------|----------|------------|
| Name | string | `name` | Yes | Kebab-case, 1-64 chars. Must match parent directory name. |
| Description | string | `description` | Yes | 1-1024 chars |
| License | string | `license` | No | SPDX identifier |
| Compatibility | []string | `compatibility` | No | Tool compatibility hints |
| Metadata | map[string]any | `metadata` | No | Freeform key-value data |
| AllowedTools | []string | `allowed-tools` | No | Tool access restrictions |

Unknown keys are rejected (FR-015). Enforced by `yaml.v3` `Decoder.KnownFields(true)`.

### IgnoreRules

The combined set of file exclusion patterns applied during packing.

| Source | Priority | Examples |
|--------|----------|---------|
| Built-in defaults | Always applied, cannot be overridden | `.git/`, `.aipkgignore`, output archive path |
| `.aipkgignore` file | User-defined, gitignore syntax | `*.log`, `build/`, `*.tmp` |

Ignore patterns are checked before artifact discovery. If a path matches, it is excluded from the archive and the `artifacts` array.

## Relationships

```
PackageManifest
  └── Artifacts []Artifact
        └── Type -> ArtifactType (enum)

Skill artifact
  └── SkillFrontmatter (parsed from SKILL.md)
        └── Name must match Artifact.Name (directory-derived)

IgnoreRules
  └── filters paths before artifact discovery
```

## State Transitions

The pack command is stateless (no persistent state). The flow is a single linear pipeline:

```
[package directory on disk]
  -> load manifest (aipkg.json)
  -> validate manifest against schema
  -> load ignore rules (.aipkgignore + defaults)
  -> discover artifacts (scan well-known dirs, apply ignore rules)
  -> validate each artifact (type-specific rules)
  -> build enriched manifest (add artifacts array)
  -> validate enriched manifest against schema
  -> create zip archive (single top-level dir)
  -> write SHA-256 sidecar
  -> [archive + sidecar on disk]
```

Failure at any validation step aborts the pipeline. No archive or sidecar is produced on failure.
